# Internal docs

Put **non end-user-facing** technical documentation here:

- Requirements notes
- Technical specifications
- Design docs
- Architecture diagrams (or links)

## Folders

- `product/`: PRD and product-level integration guides
- `db/`: Database schemas, migrations, and RLS notes
- `vendor/`: Vendor packet specs, templates, and generation requirements
- `market/`: Competitive research and positioning
